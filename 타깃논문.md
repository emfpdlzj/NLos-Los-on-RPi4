## *A review on TinyML: State‑of‑the‑art and prospects* (Partha Pratim Ray, 2021)

---

## 1) Introduction

* **문제의식**: 고성능 ML 모델의 전력·메모리 요구치 ↑, 이종(heterogeneous) 환경 격차
* **TinyML 정의**: 초저전력 **온디바이스** ML (Cloud→Embedded), 오프라인 가능·프라이버시 우수

---

## 2) Background

**요구조건**: 저전력(배터리), 대규모 센서 확장성, **수 KB~MB** 내 실행

| 항목  | 목표 수준       |
| --- | ----------- |
| 에너지 | 10–100 mAh  |
| CPU | 10–1000 MHz |
| 메모리 | ≤ 1 MB      |
| 비용  | 누적 시 부담 고려  |

**Edge 파이프라인**: 센싱 → 필터/FIFO → 로컬 추론 → 필요 시 전송

## 4) Key Enablers

* **TinyML‑as‑a‑Service**: Quantization/Pruning/Fusion 컴파일 체인
* **Hyperdimensional Computing**: 초고차원 표현 + 유사도 기반 분류
* **Swapping**: 외부 플래시와 모델 일부 교환 실행
* **Attention Condenser**: Self‑Attention으로 핵심 특성 압축
* **Constrained NAS**: HW 제약 고려 아키텍처 탐색
* **Compression**: TAC(모델 불요, 데이터 자체 압축)
* **Quantization**: QAT, **HAQ**(강화학습으로 비트폭 결정)
* **Once‑for‑All (OFA)**: PSA로 서브넷 생성 → HW별 최적
* **Benchmark**: **MLPerf Tiny** (PTQ·다기판 비교)
* **Accelerators**: SEEDoT, HBDCA, SWIS, GrAI 등

## 7) Prospects & 결론

* **과제**: HW 한계, 표준 벤치마크 부족, 인프라 제약
* **로드맵**: 모바일 오프로딩, 시스템 수준 통합
* **메시지**: TinyML은 엣지 AI의 핵심. HW‑ML 공동 설계가 관건
